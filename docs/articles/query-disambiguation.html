RAG · QUERY UNDERSTANDING

检索前，先确认用户到底在问什么

当一个知识库包含多个产品、多个版本的资料时，直接检索可能把问题带到错误的文档集合。先做对象确认和指代消歧，往往比多搜几次更重要。

学习笔记 · 依据本地《检索数据节点实现与测试》整理

一个确认节点做四件事

  1. 读取有限轮的历史会话，补足“它”“这个”等指代。
  2. 把当前问题保存下来，保证对话和排错可以追溯。
  3. 让模型提取候选对象并重写问题，输出结构化结果。
  4. 把候选对象映射到知识库标签；低置信度时请用户确认。

为什么不能默默猜

猜对时看不出问题，猜错时却会得到一本正经的错误答案。将低置信度暴露为澄清问题，是比“更像回答”的输出更可靠的产品行为。

检索质量的第一步不是相似度，而是“检索空间是否正确”。
