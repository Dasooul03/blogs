BACKEND · STREAMING

用 SSE 把工作流进度和 LLM 输出送到前端

对“服务端持续推送、客户端主要接收”的场景，SSE 是比双工协议更轻的选择：浏览器保持一个 HTTP 长连接，服务端按事件发送增量数据。

学习笔记 · 依据本地《检索 Web 服务端搭建》整理

适合什么

场景推送内容
文件导入当前节点、完成百分比、失败原因。
LLM 生成token 或文本片段增量，以及结束事件。
日志与通知持续状态变化，而不是高频双向交互。

协议最小约定

event: progress data: {"stage":"chunking","percent":42} event: answer data: {"delta":"正在生成…"}

每条消息用空行结束。前端可以按 event 分类处理进度、回答和错误；浏览器的 EventSource 也提供自动重连能力。
