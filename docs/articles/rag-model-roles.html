RAG · MODEL ROLES

RAG 里三类模型，分别做什么？

把 Embedding、Rerank 和 LLM 都叫“大模型”很容易混淆。它们在 RAG 里分工不同，也应该用不同方式评估。

学习笔记 · 依据本地《RAG 项目简介》整理

组件输入与输出主要职责
Embedding文本 → 向量把问题和资料放进可比较的语义空间，用于初步召回。
Rerank问题 + 候选片段 → 排序从较多候选里筛选更相关、更适合进入上下文的证据。
LLM问题 + 证据 → 文本组织答案、说明限制，并在证据不足时拒绝臆测。

每一层的评估问题不同

Embedding 先看正确片段是否进入候选集；Rerank 看正确片段是否被排到前面；LLM 看回答是否忠实使用了提供的上下文。把三层分开观察，才能知道问题是“没有找到”、 “找到了但没排前面”，还是“证据在但回答写错”。

模型选择不要脱离数据

中文资料、术语密集文档和多层标题的手册，对检索模型提出不同要求。先固定一组问题和来源片段，再比较模型、切片与排序策略；否则在不同数据上得到的主观感受无法真正比较。

一个小型评测表

为每个问题记录：应命中的片段、Top-K 是否命中、重排序后的名次、最终回答是否包含关键事实、是否标注来源。这个表格不需要很大，但能成为以后替换模型或调整提示词时最可靠的参照。
