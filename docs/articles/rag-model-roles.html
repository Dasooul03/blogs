RAG · MODEL ROLES

RAG 里三类模型，分别做什么？

把 Embedding、Rerank 和 LLM 都叫“大模型”很容易混淆。它们在 RAG 里分工不同，也应该用不同方式评估。

学习笔记 · 依据本地《RAG 项目简介》整理

组件输入与输出主要职责
Embedding文本 → 向量把问题和资料放进可比较的语义空间，用于召回。
Rerank问题 + 候选片段 → 排序从较多候选里筛选更相关、更适合放进上下文的证据。
LLM问题 + 证据 → 文本组织答案、说明限制，并在证据不足时拒绝臆测。

学习时不要只看最终答案

Embedding 要看正确片段有没有召回；Rerank 要看正确片段能否排到前面；LLM 要看是否忠实使用上下文。把三段分开测，调试才不会变成盲目换模型。
