RAG · INGESTION

非结构化文档，如何进入知识库？

PDF 和 Markdown 不是“读出一段文本”就完成导入了。好的导入管线要尽量保住结构、图片上下文和可用于检索的标签。

学习笔记 · 依据本地《模块流程设计》整理

任务分发 → PDF 结构化解析 / Markdown 读取 → 图片理解 → 智能切片 → 主体与标签提取 → 混合向量化 → 持久化

两个核心问题

结构会不会坍塌？

PDF 的标题、段落、表格与图片如果被拆散，后面的检索就只是在搜索残缺文本。导入阶段要尽可能把层级和图片说明一起保存。

片段能不能独立被理解？

只保存一句正文通常不够。为每个切片附加标题路径、文档类型或主题标签，能让它在脱离原文时仍保持语义。

导入的质量决定了检索的上限；不要把“文档解析”当作一个可以忽略的前置步骤。
