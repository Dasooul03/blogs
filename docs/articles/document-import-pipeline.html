RAG · INGESTION

非结构化文档，如何进入知识库？

PDF 和 Markdown 不是“读出一段文本”就完成导入了。好的导入管线要尽量保住结构、图片上下文和可用于检索的标签。

学习笔记 · 依据本地《模块流程设计》整理

任务分发 → PDF 结构化解析 / Markdown 读取 → 图片理解 → 智能切片 → 主体与标签提取 → 混合向量化 → 持久化

导入阶段真正要保护什么

文档的标题层级决定一段文字属于哪个主题；表格和图片说明可能承载正文没有出现的信息；文件名、文档类型和产品标签则决定后续能否缩小检索范围。如果这些信息在第一步被丢掉，向量库里即使有“相似文本”，也很难给出可靠答案。

输入契约

文件后缀只是第一层判断。还需要确认编码、大小、是否存在可读文本层、图片相对路径和重复导入策略。遇到扫描 PDF 或损坏文件时，流程应写入明确状态，例如“需要 OCR”或“解析失败”，而不是继续生成空切片并把错误推到检索阶段。

切片后的抽样检查

不要只看切片数量。随机挑选包含标题、表格、图片说明和跨段落描述的片段，检查它们能否独立回答“这段内容属于什么主题、来自哪里”。如果片段没有上下文，优先回到解析和切片规则，而不是急着更换向量模型。

可执行练习

选择一份 Markdown 和一份 PDF，分别保存原文、解析结果、五个随机切片及其标题路径。将这四类产物放到同一个任务目录中，后续每次改解析器都对比这些样本。
