BACKEND · FASTAPI

从一个最小 FastAPI 开始

Agent 从脚本走向服务，第一步不必复杂：一个 GET 路由、一个 POST 路由和一份明确的请求模型，就已经能把能力交给前端或其他服务调用。

学习笔记 · 依据本地《Phase 0 Day 2：FastAPI 最小 API》整理

先区分三件事

概念作用
路由定义接口路径与 HTTP 方法，例如读取状态或提交问题。
请求模型用 Pydantic 约束请求体字段，让错误在入口暴露。
响应模型让调用方知道应该得到什么结构，而不是猜 JSON。

把最小 API 用在 Agent 上

可以从 /health、/ask、/tasks/{id} 三类接口开始。前者验证服务活着，第二个接收问题，第三个为异步任务或导入流程提供可查询状态。

接口越早有清晰的数据模型，后面接 LLM、检索和流式输出时返工越少。
