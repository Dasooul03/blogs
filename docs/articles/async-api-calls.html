PYTHON · ASYNC

异步不是更快，而是别空等

调用模型、检索服务或外部 API 时，程序大部分时间都在等待网络。异步的价值是把等待时间交还给其他任务，而不是让单个请求神奇地变快。

学习笔记 · 依据本地《Phase 0 Day 3：异步编程 + 调外部 API》整理

同步等待的问题

顺序调用两个外部接口时，第一个请求等待网络的时间里，程序无法开始第二个请求。对于需要调用多个工具、同时请求 Embedding 和查询状态的 Agent 服务，这种空等会很快累积成用户可见的延迟。

三个基本规则

规则含义易错点
async def声明函数会包含可等待操作。只改函数名并不能让同步代码变快。
await等待结果并交回执行权。遗漏 await 会得到协程对象而非结果。
异步客户端用支持 async 的 HTTP 客户端发请求。在 async 函数中调用阻塞 I/O 仍会卡住服务。

在 Agent 里的使用边界

并行获取互不依赖的工具结果、请求多个 Embedding 或轮询任务状态，都是合适的异步场景。若下一步必须使用上一步的输出，就应该保留顺序。先把依赖关系画出来，再决定并发，远比“所有地方都 async”更可靠。
