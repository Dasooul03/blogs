LANGGRAPH · WORKFLOW

把文档处理写成可追踪的状态图

当流程开始包含文件分流、解析、切片、嵌入和入库时，最有价值的不是“图”，而是让每一步的输入、输出与路由都能被检查。

学习笔记 · 2026 / 07 · 预计阅读 7 分钟

先定义共享 State

State 是节点之间传递的唯一事实来源。它至少应携带任务 ID、输入路径、文档内容、切片、向量和文件类型开关；每个节点只读取自己需要的字段，并明确写回自己负责的结果。

node_entry → node_pdf_to_md → node_md_img → node_document_split → node_item_name_recognition → node_bge_embedding → node_import_milvus → END

两类边解决两类问题

边作用例子
条件边根据输入状态选择路径PDF 进入解析节点；Markdown 跳过 PDF 解析。
静态边表达确定的处理顺序切片后必须向量化；向量化后才能入库。

这套写法带来的收益

第一，输入文件不合法时可以直接结束，而不是在后续节点报模糊错误。第二，能按 task_id 追踪某一份文档经过的每个节点。第三，测试可以覆盖 PDF、Markdown 与非法格式三条路径，而不是只验证“最终有结果”。

图工作流不是为了让架构看起来复杂，而是为了把隐式控制流变成可读、可测、可恢复的显式结构。
