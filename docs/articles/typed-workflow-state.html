LANGGRAPH · STATE

为什么工作流要有显式 State？

当一个文档要经过分流、解析、切片、嵌入和入库，状态不应散落在全局变量、临时文件和函数参数里。

学习笔记 · 依据本地《导入数据图与状态定义》整理

State 里放什么

字段类别例子
追踪任务 ID、当前状态、错误信息。
路由是否为 PDF/Markdown、是否启用某种切片策略。
路径输入文件、转换后的 Markdown、索引产物位置。
处理中间结果正文、切片、向量、主体标签。

TypedDict 的小收益

用类型化的字典不只是为了代码补全。它迫使我们列出“一个节点可以读取和写入哪些数据”，进而发现默认值、缺失字段和跨平台路径问题。

State 是工作流的契约：每个节点都应该能解释自己依赖什么、产出什么，而不是暗中修改系统状态。
